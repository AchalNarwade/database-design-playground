# 🗄️ Database Design Playground

Welcome to my **Database Design Playground**!

This repository documents my journey of learning **database design** for backend development. Each project starts from a problem statement and progresses through requirements gathering, ER diagram creation, and SQL schema implementation.

The goal is to strengthen my understanding of relational databases before building backend applications with technologies like **Spring Boot** and **PostgreSQL**.

---

## 📚 Repository Structure

Each project contains:

- 📝 Problem Statement
- 📋 Requirements
- 📊 Entity Relationship Diagram (ERD)
- 🗃️ PostgreSQL Schema
- 💡 Design Notes

## 🎯 Learning Objectives

Through this repository, I aim to:

- Design normalized relational databases
- Practice identifying entities and relationships
- Understand primary and foreign keys
- Model one-to-one, one-to-many, and many-to-many relationships
- Write clean PostgreSQL schemas
- Improve database design skills for backend development
